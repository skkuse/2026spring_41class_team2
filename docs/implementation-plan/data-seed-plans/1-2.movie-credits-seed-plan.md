# 영화 출연진/제작진 Seed 적재 계획

## 범위

이 문서는 MovieLens subset으로 확정된 영화 3,000개를 기준으로 TMDB credits 데이터를 수집해 인물, 출연진, 제작진 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `people` | `people_seed.csv` | TMDB 인물 정보 |
| `movie_casts` | `movie_casts_seed.csv` | 영화별 출연진 |
| `movie_crew` | `movie_crew_seed.csv` | 영화별 제작진 |

테이블 정의는 [../../db-schema/people.md](../../db-schema/people.md)를 기준으로 한다. 전체 seed 문서 구성과 전체 CSV 적재 순서는 [1.movielens-subset-seed-plan.md](./1.movielens-subset-seed-plan.md)를 따른다.

## 선행 데이터

이 문서는 [1-1.movie-catalog-seed-plan.md](./1-1.movie-catalog-seed-plan.md)의 산출물인 `movies_seed.csv`가 확정된 뒤 진행한다.

`movie_casts.movie_id`와 `movie_crew.movie_id`는 `movies.id`를 참조하므로, credits seed는 최종 3,000개 영화에 대해서만 생성한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| `movies_seed.csv` | TMDB movie id 기준 최종 영화 목록 |
| TMDB movie credits | cast 목록, Director crew 목록 |
| TMDB person summary | `people` 보강 필드 |
| TMDB credit detail | 누락된 credit 단위 필드 보강 |

TMDB movie credits는 `/movie/{movie_id}/credits?language=ko-KR`로 조회한다. credits 응답만으로 `people`에 필요한 필드를 채울 수 있으면 별도 person detail 조회는 하지 않는다. credits 응답에 필수 필드가 없으면 필요한 row에 한해 보강 조회를 수행한다.

보강 조회 기준:

- `person_id`가 있고 `people` 필수 필드가 비어 있으면 `/person/{person_id}?language=ko-KR`를 조회한다.
- `credit_id`가 있고 cast/crew 관계 필드가 비어 있으면 `/credit/{credit_id}?language=ko-KR`를 조회한다.
- 보강 후에도 필수값이 없으면 해당 관계 row는 CSV에서 제외하고 report에 남긴다.
- `person_id` 자체가 없으면 안정적인 보강이 어렵기 때문에 검색 API로 추정 매칭하지 않는다.

## CSV 생성 규칙

### people_seed.csv

`people.id`는 TMDB person id를 그대로 사용한다. 최종 3,000개 영화의 전체 cast와 Director crew에 등장한 person을 중복 제거해 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | TMDB person id |
| `name` | TMDB person name |
| `profile_path` | TMDB profile path, 없으면 null |
| `known_for_department` | TMDB known_for_department, 없으면 null |
| `popularity` | TMDB popularity, 없으면 null |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

같은 person id가 여러 영화 또는 cast/Director crew 양쪽에 등장하면 `people_seed.csv`에는 한 번만 저장한다.

같은 person id에 대해 필드 값이 서로 다르면 다음 병합 규칙을 적용하고 report에 충돌 내역을 남긴다.

| 필드 | 병합 규칙 |
|---|---|
| `name` | 처음 확인한 non-empty 값 유지 |
| `profile_path` | 처음 확인한 non-null 값 유지 |
| `known_for_department` | 처음 확인한 non-null 값 유지 |
| `popularity` | 가장 큰 값 사용 |

### movie_casts_seed.csv

TMDB credits 응답의 `cast` 배열 전체를 기준으로 생성한다. TMDB credits에는 주연/조연/단역을 직접 구분하는 필드가 없으므로 cast 범위를 `order` 기준으로 제한하지 않는다.

| 컬럼 | 생성 규칙 |
|---|---|
| `movie_id` | `movies.id` |
| `person_id` | TMDB cast person id |
| `character_name` | TMDB cast character |
| `cast_order` | TMDB cast order, 없으면 null |

`character_name`이 비어 있는 cast row는 `credit_id`가 있으면 `/credit/{credit_id}`로 보강한다. 보강 후에도 비어 있으면 화면 표시 가치와 PK 안정성이 낮으므로 저장하지 않는다.

`person_id` 또는 `people.name`이 보강 후에도 비어 있으면 `people` FK를 안정적으로 만들 수 없으므로 해당 cast row도 저장하지 않는다. `(movie_id, person_id, character_name)` 조합을 기준으로 중복을 제거한다.

### movie_crew_seed.csv

TMDB credits 응답의 `crew` 배열 중 `job = 'Director'`인 row만 기준으로 생성한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `movie_id` | `movies.id` |
| `person_id` | TMDB crew person id |
| `department` | TMDB crew department |
| `job` | TMDB crew job |

`job` 또는 `department`가 비어 있는 crew row는 `credit_id`가 있으면 `/credit/{credit_id}`로 보강한다. 보강 후 `job = 'Director'`인 row만 저장한다.

`department`, `job`, `person_id`, `people.name` 중 필수값이 보강 후에도 비어 있으면 해당 crew row는 저장하지 않는다. `(movie_id, person_id, department, job)` 조합을 기준으로 중복을 제거한다.

감독 표시는 `job = 'Director'`를 기준으로 조회하므로 Director crew row는 누락되지 않아야 한다.

전체 crew를 저장하지 않는 이유:

- 현재 상세 화면/API 계획에서 `movie_crew`의 주 사용처는 감독 조회다.
- TMDB crew 전체에는 제작, 각본, 촬영, 편집, 음향, 미술, 의상, 조명, 시각효과 등 다양한 부서와 세부 job이 포함되지만 초기 서비스 스키마에서는 감독 외 표시 요구가 확정되어 있지 않다.
- 추후 작가, 음악감독, 촬영감독 등을 표시해야 하면 crew seed 범위를 확장한다.

## 스크립트와 산출물 경로

기존 movie catalog seed 스크립트는 다음 위치에 둔다.

```text
scripts/seeds/movie-catalog/generate-csv-from-movielens.ts
scripts/seeds/movie-catalog/upload-csv-to-supabase.ts
```

credits seed CSV 생성 스크립트는 다음 위치에 둔다.

```text
scripts/seeds/movie-credits/generate-csv-from-tmdb.ts
```

credits seed 산출물은 다음 위치에 생성한다.

```text
data/seeds/movie-credits/generated/people_seed.csv
data/seeds/movie-credits/generated/movie_casts_seed.csv
data/seeds/movie-credits/generated/movie_crew_seed.csv
```

TMDB 응답 cache와 report는 다음 위치를 사용한다.

```text
data/seeds/movie-credits/cache/
data/seeds/movie-credits/reports/generation-report.json
data/seeds/movie-credits/reports/person-conflicts.json
```

report에는 다음 내용을 포함한다.

- TMDB credits/person/credit detail 요청 수
- 보강 조회 시도/성공 수
- 보강 후에도 누락되어 제외된 cast/crew row 수와 샘플
- person 필드 충돌 내역
- cast/crew 중복 제거 수
- Director row가 없는 영화 목록
- TMDB API 실패 목록

## 적재 순서

이 문서의 CSV는 다음 순서로 적재한다.

```text
1. people_seed.csv
2. movie_casts_seed.csv
3. movie_crew_seed.csv
```

`movie_casts`와 `movie_crew`는 `movies(id)`와 `people(id)`를 모두 참조한다. `movies`는 1-1에서 먼저 적재되어 있어야 하고, `people`은 이 문서에서 cast/crew보다 먼저 적재한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| `people` 중복 | `id` 중복 0건 |
| `people.name` 누락 | 0건 |
| `movie_casts` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `movie_casts` FK | 모든 `person_id`가 `people.id`에 존재 |
| `movie_casts` 중복 | `(movie_id, person_id, character_name)` 중복 0건 |
| `movie_crew` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `movie_crew` FK | 모든 `person_id`가 `people.id`에 존재 |
| `movie_crew` 중복 | `(movie_id, person_id, department, job)` 중복 0건 |
| 감독 row | 감독이 있는 영화는 `job = 'Director'` row 존재 |

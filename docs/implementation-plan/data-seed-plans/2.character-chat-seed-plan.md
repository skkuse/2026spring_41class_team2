# 캐릭터 채팅 Seed 적재 계획

## 목적

캐릭터 채팅 seed는 캐릭터 채팅 기능에 필요한 수동 기획 데이터를 서비스 시작 전에 적재하기 위한 데이터셋이다. 이 데이터는 TMDB나 MovieLens에서 자동 생성되는 데이터가 아니라, 영화별 캐릭터 설정과 대본 기반 사건 요약을 바탕으로 작성하는 서비스 콘텐츠다.

이 문서는 캐릭터 채팅 seed 적재 계획의 상위 문서다. 캐릭터별 컬럼 매핑, 프롬프트 작성 규칙, 사건 작성 기준, 기본 질문 작성 기준, 상세 검증 기준은 하위 seed 계획 문서에서 관리한다.

테이블 정의는 [../../db-schema/character-chat.md](../../db-schema/character-chat.md)를 기준으로 한다.

## 초기 노출 범위

초기 캐릭터 채팅 기능에서 화면에 노출할 채팅 지원 영화는 2개로 고정한다. 따라서 `GET /api/character-chat/movies`는 seed된 채팅 지원 영화 전체를 반환하고, 별도의 `page`, `size` query를 받지 않는다.

지원 영화 선정 기준과 캐릭터 프로필 작성 규칙은 [2-1.character-profile-seed-plan.md](./2-1.character-profile-seed-plan.md)의 초기 Seed 대상 절을 따른다.

## 왜 별도 Seed인가

캐릭터 채팅 데이터는 MovieLens subset 파이프라인의 산출물이 아니다. MovieLens와 TMDB는 영화, 인물, 출연진의 기준 데이터로만 사용하고, 캐릭터의 말투, 인격, 배경 지식, 사건별 관점, 기본 질문은 별도로 작성한다.

영화 대본 원문은 DB에 저장하지 않는다. 대본은 `persona_prompt`, `character_chat_events`, `character_chat_event_participants`를 작성하기 위한 참고 자료로만 사용하고, DB에는 요약과 관점 데이터만 저장한다.

## Seed 대상 테이블

캐릭터 채팅 기능을 위해 미리 적재할 수동 seed 테이블은 다음 4개다.

| 그룹          | 테이블                                                       | 세부 문서                                      |
| ------------- | ------------------------------------------------------------ | ---------------------------------------------- |
| 캐릭터 프로필 | `characters`                                                 | `2-1.character-profile-seed-plan.md`           |
| 캐릭터 사건   | `character_chat_events`, `character_chat_event_participants` | `2-2.character-events-seed-plan.md`            |
| 기본 질문     | `character_chat_default_questions`                           | `2-3.character-default-questions-seed-plan.md` |

## 세부 계획 문서

| 순서 | 문서                                           | 대상 테이블                                                  | 역할                                                  |
| ---- | ---------------------------------------------- | ------------------------------------------------------------ | ----------------------------------------------------- |
| 2-1  | `2-1.character-profile-seed-plan.md`           | `characters`                                                 | 캐릭터 프로필, persona, 인사말, 이미지 경로 seed 생성 |
| 2-2  | `2-2.character-events-seed-plan.md`            | `character_chat_events`, `character_chat_event_participants` | 사건 요약과 캐릭터별 관점 seed 생성                   |
| 2-3  | `2-3.character-default-questions-seed-plan.md` | `character_chat_default_questions`                           | 캐릭터별 대화 시작 기본 질문 seed 생성                |

## 선행 데이터

캐릭터 채팅 seed는 다음 데이터가 적재된 뒤에 실행한다.

| 선행 데이터 | 이유                                                           |
| ----------- | -------------------------------------------------------------- |
| `movies`    | `characters.movie_id`, `character_chat_events.movie_id`가 참조 |
| `people`    | `characters.actor_person_id`가 선택적으로 참조                 |

`movies`와 `people`은 MovieLens subset seed 적재 계획의 산출물이다. 관련 계획은 [1.movielens-subset-seed-plan.md](./1.movielens-subset-seed-plan.md)를 따른다.

## 의존성

```text
movies
  ├─ characters
  └─ character_chat_events

people
  └─ characters

characters
  ├─ character_chat_event_participants
  └─ character_chat_default_questions

character_chat_events
  └─ character_chat_event_participants
```

## 전체 CSV 적재 순서

FK 제약을 고려한 전체 적재 순서는 다음을 따른다.

```text
characters_seed.csv
character_chat_events_seed.csv
character_chat_event_participants_seed.csv
character_chat_default_questions_seed.csv
```

`character_chat_event_participants`는 `characters`와 `character_chat_events`를 모두 참조하므로 두 CSV가 적재된 뒤 실행한다. `character_chat_default_questions`는 `characters`만 참조하므로 `characters_seed.csv` 이후 적재할 수 있다.

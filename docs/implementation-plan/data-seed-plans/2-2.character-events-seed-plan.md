# 캐릭터 사건 Seed 적재 계획

## 범위

이 문서는 캐릭터 채팅을 지원할 영화의 주요 사건과 사건별 캐릭터 관점 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `character_chat_events` | `character_chat_events_seed.csv` | 영화 안에서 실제로 일어난 객관 사건 |
| `character_chat_event_participants` | `character_chat_event_participants_seed.csv` | 사건을 각 캐릭터가 어떻게 겪고 기억하는지에 대한 관점 |

테이블 정의는 [../../db-schema/character-chat.md](../../db-schema/character-chat.md)를 기준으로 한다. 전체 캐릭터 채팅 seed 문서 구성과 전체 CSV 적재 순서는 [2.character-chat-seed-plan.md](./2.character-chat-seed-plan.md)를 따른다.

## 초기 Seed 대상

초기 캐릭터 사건 seed는 2-1에서 선정한 채팅 지원 영화 2개를 대상으로 작성한다.

영화별 사건은 8~15개로 작성한다. 사건은 장면 단위로 기계적으로 쪼개지 않고, 캐릭터 대화에서 의미 있게 회상하거나 해석할 수 있는 서사적 사건 단위로 묶는다.

사건 선정 기준은 다음과 같다.

| 기준 | 설명 |
|---|---|
| 서사 중요도 | 영화의 핵심 갈등, 선택, 반전, 관계 변화를 설명하는 사건이다. |
| 대화 활용도 | 사용자가 캐릭터에게 질문했을 때 답변 근거로 사용할 수 있다. |
| 캐릭터 관점 차이 | 같은 사건에 대해 캐릭터마다 다른 감정, 해석, 지식 상태를 가질 수 있다. |
| 지식 상태 변화 | 사건 이후 캐릭터가 알게 되는 사실이나 여전히 모르는 사실이 분명하다. |
| 대표 캐릭터 연결성 | 2-1에서 seed한 대표 캐릭터 중 최소 1명 이상과 관련된다. |

## 선행 데이터

이 문서는 다음 데이터가 확정된 뒤 진행한다.

| 선행 데이터 | 이유 |
|---|---|
| `movies_seed.csv` | `character_chat_events.movie_id`가 `movies.id`를 참조한다. |
| `characters_seed.csv` | `character_chat_event_participants.character_id`가 `characters.id`를 참조한다. |
| 영화별 `characters.config.ts` | participant의 `characterName`을 2-1 캐릭터 목록과 대조한다. |

`character_chat_events_seed.csv`는 `characters_seed.csv` 없이도 생성할 수 있지만, `character_chat_event_participants_seed.csv`는 2-1 캐릭터 ID 생성 규칙과 캐릭터 목록에 의존한다. 따라서 실제 검증과 적재는 `characters_seed.csv` 생성 이후 수행한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| 영화별 `events.config.ts` | 사건 제목, 순서, 객관 요약, 캐릭터별 관점 |
| 영화별 `characters.config.ts` | participant의 `characterName` 검증과 `character_id` 계산 |
| `movies_seed.csv` | `movie_id` FK 검증 |
| `characters_seed.csv` | `character_id` FK 검증 |

원천 파일 구조는 다음을 따른다.

```text
data/seeds/character-chat/movies/{movie_id}/characters.config.ts
data/seeds/character-chat/movies/{movie_id}/events.config.ts
data/seeds/character-chat/generated/character_chat_events_seed.csv
data/seeds/character-chat/generated/character_chat_event_participants_seed.csv
scripts/seeds/generate-character-event-seed.ts
```

사람이 관리하는 원천 데이터는 영화별 `movies/{movie_id}` 하위에 둔다. `generated/*.csv`는 스크립트 산출물이므로 직접 편집하지 않는다.

## 영화별 Events Config 규칙

`events.config.ts`는 해당 영화의 주요 사건과 사건별 캐릭터 관점을 정의한다.

기본 형태:

```ts
export const characterChatEvents = {
  movieId: 603,
  events: [
    {
      order: 1,
      title: "네오가 정체불명의 메시지를 받는다",
      summary:
        "네오는 자신의 컴퓨터를 통해 알 수 없는 메시지를 받고, 현실에 대한 의문을 더 강하게 품기 시작한다.",
      participants: [
        {
          characterName: "Neo",
          role: "subject",
          perspectiveSummary:
            "네오는 이 사건을 자신이 찾던 답으로 향하는 첫 신호처럼 받아들인다.",
          emotionalImpact:
            "불안과 호기심이 동시에 커지고, 평범한 일상으로 돌아가기 어렵다고 느낀다.",
          knowledgeState:
            "네오는 메시지를 보낸 존재와 세계의 진짜 구조를 아직 알지 못한다.",
        },
      ],
    },
  ],
} as const
```

필수 필드는 다음과 같다.

| 필드 | 설명 |
|---|---|
| `movieId` | TMDB movie id. `movies.id`와 일치해야 한다. |
| `events[].order` | 영화 안의 사건 순서. 사람이 직접 1부터 작성한다. |
| `events[].title` | 사건 짧은 제목. 한국어 30자 안팎으로 작성한다. |
| `events[].summary` | 객관 사건 요약. 한국어 1~3문장으로 작성한다. |
| `events[].participants` | 해당 사건과 관련된 캐릭터별 관점 목록. 최소 1명 이상이어야 한다. |
| `participants[].characterName` | 2-1 `characters.config.ts`의 캐릭터 이름과 일치해야 한다. |
| `participants[].role` | 사건에서 캐릭터의 역할. 자유 텍스트이되 짧은 label을 권장한다. |
| `participants[].perspectiveSummary` | 캐릭터 관점의 사건 요약. 한국어 1~3문장으로 작성한다. |
| `participants[].emotionalImpact` | 사건이 캐릭터에게 남긴 감정과 변화. 한국어 1~2문장으로 작성한다. |
| `participants[].knowledgeState` | 이 사건 직후 캐릭터가 알고 있거나 모르는 사실. 한국어 1~3문장으로 작성한다. |

`role`은 DB와 CSV에서는 자유 텍스트로 허용한다. 다만 검색과 검수 편의를 위해 `subject`, `witness`, `helper`, `opponent`, `betrayer`, `betrayed`, `victim`, `cause`, `observer`, `absent-informed`처럼 짧은 label 형태를 권장한다. 긴 설명은 `perspectiveSummary`에 작성한다.

## 사건 작성 규칙

사건은 실제 영화 안에서 일어난 일을 객관적으로 요약한다.

`summary`에는 캐릭터별 감정, 해석, 내면 독백을 넣지 않는다. 감정과 해석은 `character_chat_event_participants`의 `perspectiveSummary`, `emotionalImpact`, `knowledgeState`에 작성한다.

영화 대본 원문, 긴 원문 인용, 원문 대사 번역은 저장하지 않는다. 대본과 장면은 참고 자료로만 사용하고, DB에는 요약과 관점 데이터만 저장한다.

사용자는 영화를 이미 본 사람으로 가정하므로 `events.config.ts`에는 결말, 반전, 핵심 정체성 변화 같은 스포일러를 포함할 수 있다.

## Participant 작성 규칙

participant는 사건과 관련 있는 캐릭터만 작성한다. 모든 사건에 모든 대표 캐릭터 row를 만들지 않는다.

다음 경우 participant로 작성할 수 있다.

| 경우 | 설명 |
|---|---|
| 직접 참여 | 캐릭터가 사건 안에서 행동하거나 선택한다. |
| 목격 | 캐릭터가 사건을 직접 보고 듣는다. |
| 영향 | 캐릭터가 직접 등장하지 않아도 사건의 영향을 크게 받는다. |
| 사후 인지 | 캐릭터가 사건 당시에는 없었지만 나중에 사건을 알게 된다. |

사건 당시 직접 보지 않았지만 나중에 알게 된 캐릭터는 participant로 작성할 수 있다. 이 경우 `role`에는 `absent-informed` 같은 짧은 label을 사용할 수 있고, `knowledgeState`에 언제 무엇을 알게 되었는지와 여전히 모르는 사실을 명확히 작성한다.

`knowledgeState`는 사건 직후 기준으로 작성한다. 캐릭터가 영화 안에서 끝까지 알 수 없는 사실은 안다고 쓰지 않는다. 캐릭터가 모르는 사실을 질문받았을 때 추측, 의심, 부정, 모름으로 답할 수 있도록 제한하는 정보가 되어야 한다.

## Character Slug 참조 규칙

`events.config.ts`에는 `characterSlug`나 `character_id`를 직접 쓰지 않는다. 사람이 읽기 쉬운 `characterName`만 작성한다.

생성 스크립트는 2-1과 같은 slug 규칙으로 `characterName`을 정규화하고, 해당 영화의 `characters.config.ts`에 같은 이름의 캐릭터가 있는지 검증한다.

정규화된 `character_slug`는 다음 용도로만 사용한다.

```text
1. 2-1과 같은 deterministic character UUID v5 계산
2. participant의 character_id 생성
3. characterName 오타와 중복 검증
```

2-1 캐릭터 목록에 없는 `characterName`이 participant에 있으면 seed 생성은 실패한다.

## Event Slug 규칙

`event_slug`는 DB 컬럼이나 CSV 컬럼으로 저장하지 않는다. 생성 스크립트가 `title`을 정규화해 내부적으로 사용한다.

정규화 규칙은 2-1의 `character_slug`와 동일하게 적용한다.

```text
소문자로 변환한다.
공백과 특수문자는 하이픈으로 변환한다.
연속 하이픈은 하나로 줄인다.
앞뒤 하이픈은 제거한다.
영화 안에서 유일해야 한다.
```

같은 영화 안에서 생성된 `event_slug`가 중복되면 seed 생성은 실패한다.

## CSV 생성 규칙

`character_chat_events_seed.csv`와 `character_chat_event_participants_seed.csv`는 모든 영화별 `events.config.ts`를 읽어 생성한다.

### character_chat_events_seed.csv

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | `movie_id + event_order + event_slug` 기반 deterministic UUID v5 |
| `movie_id` | `events.config.ts`의 `movieId` |
| `event_order` | `events[].order` |
| `title` | `events[].title` |
| `summary` | `events[].summary` |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

`id`는 같은 `movie_id`, `event_order`, `event_slug`에 대해 항상 같은 값이 생성되어야 한다. UUID namespace는 생성 스크립트에 상수로 고정한다.

UUID v5 입력 문자열은 다음 형식을 따른다.

```text
character-chat-event:{movie_id}:{event_order}:{event_slug}
```

### character_chat_event_participants_seed.csv

| 컬럼 | 생성 규칙 |
|---|---|
| `event_id` | 같은 event row의 deterministic UUID v5 |
| `character_id` | 2-1과 같은 `movie_id + character_slug` 기반 deterministic UUID v5 |
| `role` | `participants[].role` |
| `perspective_summary` | `participants[].perspectiveSummary` |
| `emotional_impact` | `participants[].emotionalImpact` |
| `knowledge_state` | `participants[].knowledgeState` |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

`character_chat_event_participants`는 `(event_id, character_id)` 복합 PK를 사용하므로 별도 ID를 생성하지 않는다.

## 생성 스크립트 규칙

생성 스크립트는 다음 경로에 둔다.

```text
scripts/seeds/generate-character-event-seed.ts
```

스크립트는 다음 작업을 수행한다.

| 단계 | 설명 |
|---|---|
| config 로드 | 모든 영화별 `events.config.ts`와 `characters.config.ts`를 읽는다. |
| 영화 검증 | `movieId`가 `movies_seed.csv`에 존재하는지 확인한다. |
| 캐릭터 검증 | participant의 `characterName`이 같은 영화의 2-1 캐릭터 목록에 존재하는지 확인한다. |
| event slug 생성 | `title`에서 `event_slug`를 계산한다. |
| event UUID 생성 | `character-chat-event:{movie_id}:{event_order}:{event_slug}`로 UUID v5를 생성한다. |
| character UUID 생성 | 2-1과 같은 규칙으로 participant의 `character_id`를 생성한다. |
| CSV 출력 | events CSV와 participants CSV를 `generated/` 아래에 생성한다. |

스크립트는 검증 실패가 있으면 CSV를 생성하지 않는다. 이미 생성된 CSV가 있더라도 실패한 입력으로 덮어쓰지 않는다.

## 적재 순서

이 문서의 CSV는 `characters_seed.csv` 생성 이후 만들고 적재한다.

```text
1. characters_seed.csv
2. generate-character-event-seed.ts
3. character_chat_events_seed.csv
4. character_chat_event_participants_seed.csv
```

`character_chat_event_participants_seed.csv`는 `characters_seed.csv`와 `character_chat_events_seed.csv`를 모두 참조하므로 두 CSV가 준비된 뒤 적재한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| 초기 영화 수 | 2-1과 같은 채팅 지원 영화 2개 |
| 영화별 사건 수 | 8~15개. 범위를 벗어나면 실패 |
| 사건 단위 | 서사적으로 의미 있는 사건 단위 |
| `movie_id` FK | 모든 `movieId`가 `movies.id`에 존재 |
| `event_order` | 영화별로 1부터 중복 없이 작성 |
| `event_slug` 중복 | 같은 영화 안에서 생성된 slug 중복 0건 |
| `events.id` | UUID v5 재생성 결과와 CSV의 `id`가 일치 |
| `title` | 비어 있지 않고 한국어 30자 안팎 |
| `summary` | 비어 있지 않고 한국어 1~3문장 |
| participant 수 | 모든 사건에 participant 1명 이상 |
| `characterName` | 같은 영화의 2-1 캐릭터 목록에 존재 |
| participant 중복 | 같은 사건 안에서 같은 캐릭터 중복 0건 |
| `character_id` FK | 모든 `character_id`가 `characters.id`에 존재 |
| `role` | 비어 있지 않음 |
| `perspective_summary` | 비어 있지 않고 한국어 1~3문장 |
| `emotional_impact` | 비어 있지 않고 한국어 1~2문장 |
| `knowledge_state` | 비어 있지 않고 한국어 1~3문장 |
| 원문 저장 금지 | 영화 대본 원문, 긴 원문 인용, 긴 번역문을 포함하지 않음 |
| timestamp 컬럼 | `created_at`, `updated_at`은 CSV에 포함하지 않음 |

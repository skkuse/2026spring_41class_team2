# Cinemate 서비스 정책

## 인증 및 사용자 정책

- 인증 방식은 Supabase Auth를 사용한다.
- 비로그인 사용자는 영화 탐색, 검색, 상세 정보 조회만 가능하다.
- 비로그인 사용자는 AI 영화 추천 채팅, 캐릭터 채팅, 맞춤 추천(`/recommend`)을 사용할 수 없다.
- 비로그인 상태에서 영화 목록/상세의 `isLiked`는 항상 `false`로 처리한다.
- 비로그인 사용자가 찜 버튼이나 리뷰 작성 버튼을 누르면 로그인 페이지로 보낸다.

## 온보딩 정책

- 비로그인 온보딩은 허용하지 않는다.
- 온보딩은 회원가입 후 온보딩을 완료하지 않은 로그인 사용자만 접근한다.
- 온보딩 완료 후에는 온보딩 화면으로 다시 넘어가지 않는다.
- 온보딩에서는 장르가 아니라 인기 영화 목록에서 선호 영화 5개를 선택한다.
- 온보딩 영화 선택은 정확히 5개를 선택해야 완료할 수 있다.
- 온보딩 영화 후보는 MovieLens 매핑이 있는 인기 영화로 한정한다.
- 온보딩에서는 영화 검색을 제공하지 않는다.
- 온보딩에서 선택한 영화는 `/recommend` Item CF 추천의 seed로 사용한다.

## 영화 데이터 소스

- 영화 데이터 소스는 MovieLens + TMDB enrich 방식으로 간다.
- MovieLens는 추천 학습 및 랭킹용 데이터로 사용한다.
- TMDB는 영화 상세 정보 원천으로 사용한다.
- MovieLens `links.csv`의 `tmdbId`를 사용해 MovieLens 영화와 TMDB 영화를 연결한다.
- IMDB는 사용하지 않는다.
- 영화 적재 범위는 MovieLens에 `tmdbId`가 있는 영화만으로 한다.
- TMDB 전체 영화를 적재하지 않는다.
- 영화 동기화는 최초 적재 후 고정으로 한다.
- 검색(`/search`)은 기본적으로 우리 DB에서 검색한다.
- TMDB 실시간 검색은 기본 검색 정책에 포함하지 않는다.

## 영화 식별자 및 메타데이터

- 영화 기본 ID는 TMDB movie id를 사용한다.
- `movies.id`는 TMDB movie id이며, DB 주석과 문서에 명확히 남긴다.
- 다른 테이블에서 영화 FK는 `movie_id`로 두고 `movies.id`를 참조한다.
- 장르는 TMDB 장르 기준으로 통일한다.
- MovieLens 장르는 화면과 검색의 장르 기준으로 사용하지 않는다.
- 배우와 감독 정보는 TMDB에서 가져온다.
- 출연진과 제작진은 분리해서 관리한다.

## 평점 정책

- 평점은 TMDB 평점을 사용하지 않는다.
- 화면 카드/상세의 기본 평점은 MovieLens 평균 평점과 Cinemate 리뷰 평점의 합산 평균으로 계산한다.
- `display_rating` 같은 계산 결과는 DB에 저장하지 않는다.
- API 호출 시점에 평점을 계산해서 응답한다.
- 리뷰 작성 시에는 `movies.cinemate_rating_sum`과 `movies.cinemate_review_count`만 갱신한다.

## 리뷰 및 찜 정책

- 리뷰는 한 유저가 한 영화에 하나만 작성할 수 있다.
- 리뷰 수정/삭제는 일단 배제한다.
- 리뷰 목록은 공개 조회 가능하다.
- 리뷰 작성과 리뷰 좋아요는 로그인 사용자만 가능하다.
- 찜 기능은 로그인 사용자만 가능하다.
- 찜 추가/삭제는 사용자-영화 조인 테이블로 관리한다.

## 추천 데이터 처리

- `/recommend` 맞춤 추천은 User CF가 아니라 MovieLens 기반 Item CF로 구현한다.
- MovieLens 원본/전처리 데이터는 Python 추천 서버에서 사용한다.
- 백엔드는 로그인 사용자와 온보딩 seed 영화, 제외할 영화 목록을 구성한 뒤 Python 추천 서버를 호출한다.
- Python 추천 서버는 seed 영화별 Item CF 추천 결과를 반환한다.
- `/recommend` 추천 결과는 DB에 미리 저장하지 않는다.
- `seedLimit`은 온보딩에서 선택한 영화 5개 중 추천 섹션을 만들 기준 영화를 몇 개 사용할지 의미한다.
- `limitPerSeed`는 각 seed 영화 섹션마다 추천 영화를 몇 개 반환할지 의미한다.
- `excludeMovieIds`는 온보딩에서 선택한 영화, 이미 찜한 영화, 이미 리뷰한 영화처럼 최종 추천 결과에서 제외할 영화 ID 목록이다.
- 추천 결과가 부족하면 평점 높은 영화로 보충한다.
- `/recommend` 화면 섹션 제목은 `{영화 제목}을 좋아한 사람들이 함께 좋아한 영화` 형식을 사용한다.
- 추천 결과의 `score`, `reason`은 Python 추천 서버 결과와 백엔드 템플릿 조합으로 응답한다.

## 채팅 도메인 분리

- AI 영화 추천 채팅과 캐릭터 채팅은 분리한다.
- AI 영화 추천 채팅 테이블 이름은 `recommendation_chat_*`로 정한다.
- AI 영화 추천 채팅 결과는 메시지 단위로 저장한다.
- 캐릭터 채팅은 `character_chat_*` 테이블로 분리한다.
- 캐릭터 대화는 AI 추천 채팅과 별도 도메인으로 관리한다.
- 캐릭터 대본/설정은 수동으로 관리한다.
- 캐릭터 데이터는 TMDB 자동 파생 데이터가 아니라 서비스 기획 데이터로 본다.
- 캐릭터는 특정 영화와 연결된다.
- 캐릭터는 필요하면 TMDB 배우 정보와 연결할 수 있다.

## RLS 정책

- RLS 정책은 서비스 정책에 맞게 설정한다.
- 영화, 장르, 배우/인물 정보는 공개 읽기 가능하게 한다.
- 프로필은 본인만 수정 가능하게 한다.
- 찜은 본인 데이터만 접근 가능하게 한다.
- 추천 채팅은 본인 대화만 접근 가능하게 한다.
- 캐릭터 채팅은 본인 대화만 접근 가능하게 한다.
- 리뷰 목록은 공개 조회 가능하게 한다.
- 리뷰 작성과 리뷰 좋아요는 로그인 사용자만 가능하게 한다.

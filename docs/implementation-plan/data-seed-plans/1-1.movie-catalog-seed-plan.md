# 영화 카탈로그 Seed 적재 계획

## 범위

이 문서는 MovieLens subset으로 확정된 영화 3,000개를 기준으로 영화 카탈로그 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `movies` | `movies_seed.csv` | TMDB 기반 영화 카탈로그 |
| `genres` | `genres_seed.csv` | TMDB 장르 마스터 |
| `movie_genres` | `movie_genres_seed.csv` | 영화와 장르의 다대다 연결 |

테이블 정의는 [../../db-schema/movies.md](../../db-schema/movies.md)를 기준으로 한다. 전체 seed 문서 구성과 전체 CSV 적재 순서는 [1.movielens-subset-seed-plan.md](./1.movielens-subset-seed-plan.md)를 따른다.

## 후보 선정 기준

기본 후보는 다음 hard 조건을 모두 만족해야 한다. hard 조건은 최종 3,000개 보장을 위해서도 완화하지 않는다.

```text
links.csv에 tmdbId가 존재한다.
tmdbId가 중복되지 않는다.
TMDB movie detail 조회에 성공한다.
poster_path가 존재한다.
ko-KR 또는 en-US overview가 존재한다.
MovieLens rating이 1개 이상 있어 movielens_avg_rating을 계산할 수 있다.
```

`movielens_rating_count`와 `user_tag_count`는 후보 품질을 높이기 위한 soft 조건이다. 최종 3,000개를 보장하기 위해 다음 bucket 순서로 후보를 확장한다.

| bucket | 조건 |
|---|---|
| 0 | `movielens_rating_count >= 500` and `user_tag_count >= 20` |
| 1 | `movielens_rating_count >= 500` |
| 2 | `movielens_rating_count >= 250` |
| 3 | `movielens_rating_count >= 100` |
| 4 | `movielens_rating_count >= 50` |
| 5 | `movielens_rating_count >= 1` |

각 bucket 안의 후보 영화는 다음 점수로 정렬한다.

```text
selection_score =
  log(movielens_rating_count + 1) * 0.6
  + log(user_tag_count + 1) * 0.3
  + movielens_avg_rating * 0.1
```

평점 수는 대중적 신뢰도, 사용자 태그 수는 메타데이터 풍부함, 평균 평점은 품질 보정 신호로 사용한다.

최종 3,000개는 반드시 보장한다. hard 조건 미충족으로 제외되는 영화가 생기면 같은 bucket의 다음 순위 영화로 먼저 backfill하고, 해당 bucket 후보가 부족하면 다음 bucket으로 확장한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| MovieLens `links.csv` | `movielens_id`, `tmdb_id` 매핑 |
| MovieLens `ratings.csv` | `movielens_rating_count`, `movielens_avg_rating` 계산 |
| MovieLens `tags.csv` | `user_tag_count` 계산 |
| TMDB movie detail | 영화 기본 정보, 제작 국가, 장르 |
| TMDB movie videos | 예고편 URL 후보 |
| TMDB genre list | 장르 영문명, 한국어명 |

TMDB 조회는 `ko-KR`을 우선 사용하고, 필수 표시 데이터가 비어 있으면 `en-US` 응답으로 fallback한다.

## 후보 제외 기준

다음 중 하나라도 해당하면 해당 영화는 제외하고 다음 순위 후보로 backfill한다.

```text
links.csv에 tmdbId 없음
tmdbId 중복
TMDB 조회 실패
poster_path 없음
ko-KR와 en-US overview 모두 없음
MovieLens rating 없음
```

최종 `movies_seed.csv`는 정확히 3,000개 row를 생성해야 한다. bucket 5까지 확장해도 hard 조건을 만족하는 영화가 3,000개 미만이면 seed 생성을 실패 처리하고, 부족한 row 수와 후보 제외 사유별 집계가 로그에 남아야 한다.

## CSV 생성 규칙

### movies_seed.csv

`movies.id`는 TMDB movie id를 그대로 사용한다. `movielens_id`는 MovieLens 원본 `movieId`를 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | TMDB `id` |
| `movielens_id` | MovieLens `movieId` |
| `title` | `ko-KR` title 우선, 없으면 `en-US` title |
| `original_title` | TMDB `original_title` |
| `overview` | `ko-KR` overview 우선, 없으면 `en-US` overview |
| `release_date` | TMDB `release_date`, 비어 있으면 null |
| `release_year` | `release_date`의 연도, `release_date`가 없으면 null |
| `runtime` | TMDB `runtime`, 비어 있으면 null |
| `original_language` | TMDB `original_language` |
| `production_countries` | TMDB `production_countries[].iso_3166_1` 값만 JSON 배열로 저장 |
| `poster_path` | TMDB `poster_path` |
| `backdrop_path` | TMDB `backdrop_path`, 비어 있으면 null |
| `trailer_url` | TMDB videos에서 선택한 예고편 URL, 없으면 null |
| `adult` | TMDB `adult`, 비어 있으면 false |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

`title`과 `overview`는 필드별로 fallback한다. 예를 들어 `ko-KR` title이 있고 `ko-KR` overview가 비어 있으면 title은 한국어 값을 쓰고 overview만 `en-US` 값을 사용한다.

예고편 URL은 다음 우선순위로 선택한다.

1. `site = YouTube`, `type = Trailer`, `official = true`
2. `site = YouTube`, `type = Trailer`
3. 조건을 만족하는 항목이 없으면 null

YouTube 영상은 `https://www.youtube.com/watch?v={key}` 형식으로 저장한다.

### genres_seed.csv

`genres`는 최종 3,000개 영화에 연결된 TMDB 장르만 중복 제거해 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | TMDB genre id |
| `name` | TMDB `en-US` 장르명 |
| `name_ko` | TMDB `ko-KR` 장르명, 없으면 `name` |

MovieLens 장르는 저장하지 않는다.

### movie_genres_seed.csv

TMDB movie detail 응답의 `genres` 배열을 기준으로 생성한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `movie_id` | `movies.id` |
| `genre_id` | TMDB genre id |

`movie_id`, `genre_id` 조합은 중복 없이 한 번만 저장한다.

## 적재 순서

이 문서의 CSV는 다음 순서로 적재한다.

```text
1. genres_seed.csv
2. movies_seed.csv
3. movie_genres_seed.csv
```

`genres`와 `movies`는 서로 FK 의존성이 없다. `movie_genres`는 `movies(id)`와 `genres(id)`를 모두 참조하므로 마지막에 적재한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| `movies` row 수 | 3,000개 |
| 중복 `movies.id` | 0건 |
| 중복 `movies.movielens_id` | 0건 |
| `movies.poster_path` 누락 | 0건 |
| `movies.overview` 누락 | 0건 |
| `movies.production_countries` 형식 | JSON 문자열 배열이며 각 값은 TMDB `iso_3166_1` 국가 코드 |
| `genres` row 수 | 최종 영화에 연결된 TMDB 장르 수와 일치 |
| `movie_genres` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `movie_genres` FK | 모든 `genre_id`가 `genres.id`에 존재 |
| `movie_genres` 중복 | `(movie_id, genre_id)` 중복 0건 |

# 캐릭터 프로필 Seed 적재 계획

## 범위

이 문서는 캐릭터 채팅을 지원할 영화의 대표 캐릭터 프로필 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `characters` | `characters_seed.csv` | 캐릭터 프로필, persona prompt, 인사말, 이미지 경로 |

테이블 정의는 [../../db-schema/character-chat.md](../../db-schema/character-chat.md)를 기준으로 한다. 전체 캐릭터 채팅 seed 문서 구성과 전체 CSV 적재 순서는 [2.character-chat-seed-plan.md](./2.character-chat-seed-plan.md)를 따른다.

## 초기 Seed 대상

초기 캐릭터 채팅 seed는 MovieLens subset 3,000개 영화 중 채팅 지원 영화 2개를 선정해 작성한다.

채팅 지원 영화는 다음 기준을 고려해 선정한다.

| 기준 | 설명 |
|---|---|
| 캐릭터 선명도 | 주요 인물의 말투, 욕망, 가치관이 서로 구분된다. |
| 대화 소재 | 사건, 선택, 갈등, 관계에 대해 사용자가 물어볼 거리가 충분하다. |
| 대표성 | 데모 사용자가 영화와 캐릭터를 알아보기 쉽다. |
| 사건 구조 | 2-2 사건 seed로 나눌 주요 사건이 충분하다. |
| 캐릭터 수 적합성 | 대표 캐릭터 2~4명만으로 캐릭터 채팅 경험이 성립한다. |
| 데이터 연결성 | `movies`, `people` seed 데이터와 연결하기 쉽다. |

실제 선정된 영화와 캐릭터 목록은 문서에 고정하지 않고 영화별 `characters.config.ts`에서 관리한다.

## 선행 데이터

이 문서는 다음 데이터가 확정된 뒤 진행한다.

| 선행 데이터 | 이유 |
|---|---|
| `movies_seed.csv` | `characters.movie_id`가 `movies.id`를 참조한다. |
| `people_seed.csv` | `characters.actor_person_id`가 선택적으로 `people.id`를 참조한다. |
| 캐릭터 이미지 파일 | 초기 seed 캐릭터는 전용 이미지 업로드를 완료한 뒤 CSV를 생성한다. |

`actor_person_id`는 nullable이다. 값이 있으면 반드시 `people.id`에 존재해야 한다. 캐릭터와 배우가 명확하지만 해당 배우가 `people`에 없으면 `actor_person_id`를 null로 둘 수 있고, 생성 스크립트는 작성자가 알 수 있도록 경고를 출력한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| 영화별 `characters.config.ts` | 캐릭터 이름, 배우 ID, 설명, 인사말, 이미지 object path |
| 캐릭터별 로컬 이미지 파일 | Supabase Storage 업로드 원본 |
| 캐릭터별 persona prompt Markdown | `characters.persona_prompt`에 저장할 캐릭터 대화 프롬프트 |
| Supabase Storage `character-images` bucket | 캐릭터 전용 이미지 object 존재 여부 검증 |
| `movies_seed.csv` | `movie_id` FK 검증 |
| `people_seed.csv` | `actor_person_id` FK 검증 |

원천 파일 구조는 다음을 따른다.

```text
data/seeds/character-chat/movies/{movie_id}/characters.config.ts
data/seeds/character-chat/movies/{movie_id}/images/characters/{character_slug}.webp
data/seeds/character-chat/movies/{movie_id}/prompts/characters/{character_slug}.md
data/seeds/character-chat/generated/characters_seed.csv
scripts/seeds/upload-character-images.ts
scripts/seeds/generate-character-profile-seed.ts
```

사람이 관리하는 원천 데이터는 영화별 `movies/{movie_id}` 하위에 둔다. `generated/characters_seed.csv`는 스크립트 산출물이므로 직접 편집하지 않는다.

## 영화별 Config 규칙

`characters.config.ts`는 해당 영화의 대표 캐릭터 메타데이터를 정의한다.

필수 필드는 다음과 같다.

| 필드 | 설명 |
|---|---|
| `movieId` | TMDB movie id. `movies.id`와 일치해야 한다. |
| `name` | 캐릭터 이름. 영화 안에서 중복되지 않아야 한다. |
| `actorPersonId` | TMDB person id 또는 null. 값이 있으면 `people.id`에 존재해야 한다. |
| `description` | UI에 노출할 짧은 캐릭터 소개. |
| `greeting` | 채팅 시작 시 표시할 첫 인사. |
| `avatarStoragePath` | Supabase Storage `character-images` bucket의 object path. |

`description`은 한국어 1~2문장으로 작성하고, 반전, 결말, 핵심 정체성 변화 같은 스포일러를 포함하지 않는다. 스포일러성 정보는 `persona_prompt`와 2-2 사건/관점 데이터에서만 다룬다.

`greeting`은 한국어 1~2문장으로 작성한다. 캐릭터 원작의 말투와 분위기를 한국어로 재현하되, 영화 원문 대사나 원문 번역을 길게 저장하지 않는다.

## Character Slug 규칙

`character_slug`는 DB 컬럼이나 CSV 컬럼으로 저장하지 않는다. 생성 스크립트가 `name`을 정규화해 내부적으로 사용한다.

`character_slug`는 다음 용도로만 사용한다.

```text
1. deterministic UUID v5 생성 입력값
2. persona prompt Markdown 경로 계산
3. avatar_storage_path 형식 검증
```

정규화 규칙:

```text
소문자로 변환한다.
공백과 특수문자는 하이픈으로 변환한다.
연속 하이픈은 하나로 줄인다.
앞뒤 하이픈은 제거한다.
영화 안에서 유일해야 한다.
```

예시:

```text
Neo -> neo
Agent Smith -> agent-smith
Dr. Ellie Sattler -> dr-ellie-sattler
```

같은 영화 안에서 생성된 `character_slug`가 중복되면 seed 생성은 실패한다.

## Persona Prompt 규칙

캐릭터별 persona prompt는 다음 경로의 Markdown 파일로 작성한다.

```text
data/seeds/character-chat/movies/{movie_id}/prompts/characters/{character_slug}.md
```

이 파일의 전체 본문이 `characters.persona_prompt`에 저장된다.

persona prompt는 캐릭터의 지속적인 정체성, 말투, 성격, 기본 배경, 응답 규칙을 담는다. 영화 내 구체 사건과 사건 이후 지식 상태는 2-2의 `character_chat_events`, `character_chat_event_participants`를 우선한다. 같은 사건 내용을 persona prompt에 장문으로 반복하지 않는다.

사용자는 영화를 이미 본 사람으로 가정하므로 스포일러 자체를 숨기지 않는다. 다만 캐릭터는 자기 시점에서 알 수 없는 사실을 알면 안 된다. 캐릭터가 모르는 사실을 질문받으면 추측, 의심, 부정, 모름으로 답해야 한다.

persona prompt Markdown은 다음 섹션을 모두 포함해야 한다.

```markdown
## 역할
## 말투
## 성격
## 배경 지식
## 알고 있는 사실
## 모르는 사실
## 응답 규칙
## 금지 사항
## 예시 응답
```

`예시 응답`은 반드시 한국어로 새로 작성한다. 영화 원문 대사나 원문 대사의 번역을 저장하지 않는다. 캐릭터의 말투, 태도, 어휘감을 반영한 짧은 응답 예시를 작성하되, 2-2 사건별 지식 상태와 충돌하지 않아야 한다.

## 캐릭터 이미지 규칙

초기 seed 대상 캐릭터는 전용 이미지가 먼저 준비되어야 한다. 사용자가 캐릭터별 이미지 파일을 제공하면 Supabase Storage `character-images` bucket에 업로드한 뒤, 확인된 object path를 `characters.config.ts`의 `avatarStoragePath`에 기록한다.

이미지는 `webp`로 통일한다.

object path는 다음 형식을 따른다.

```text
movies/{movie_id}/characters/{character_slug}.webp
```

예시:

```text
movies/603/characters/neo.webp
```

초기 seed 작성 기준과 DB 스키마에서는 모든 캐릭터가 `avatarStoragePath`를 가져야 한다. 배우 프로필 이미지는 캐릭터 전용 이미지의 대체 데이터로 사용하지 않는다.

## 이미지 업로드 스크립트 규칙

캐릭터 이미지 업로드는 CSV 생성보다 먼저 실행한다.

```text
scripts/seeds/upload-character-images.ts
```

업로드 스크립트는 모든 영화별 `characters.config.ts`를 읽고, 각 캐릭터 이름에서 계산한 `character_slug`를 기준으로 로컬 이미지 파일을 찾는다.

```text
data/seeds/character-chat/movies/{movie_id}/images/characters/{character_slug}.webp
```

업로드 대상 bucket과 object path는 다음과 같이 고정한다.

```text
bucket: character-images
object path: movies/{movie_id}/characters/{character_slug}.webp
```

스크립트는 다음 작업을 수행한다.

| 단계 | 설명 |
|---|---|
| config 로드 | 영화별 `characters.config.ts`를 읽는다. |
| slug 생성 | `name`에서 `character_slug`를 계산한다. |
| 로컬 파일 검증 | `{movie_id}/images/characters/{character_slug}.webp` 파일이 존재하고 확장자가 `webp`인지 확인한다. |
| path 검증 | `avatarStoragePath`가 `movies/{movie_id}/characters/{character_slug}.webp`와 일치하는지 확인한다. |
| Supabase 업로드 | `character-images` bucket에 같은 object path로 업로드한다. |
| 업로드 확인 | 업로드 후 object가 bucket에 존재하는지 확인한다. |

Supabase 업로드에는 서버 권한 key를 사용한다. 필요한 환경 변수 이름은 구현 시점에 프로젝트의 Supabase 설정 규칙을 따르되, 브라우저 공개 key로 업로드하지 않는다.

기본 동작은 같은 object path가 이미 있으면 실패하도록 한다. 기존 이미지를 갱신해야 하는 경우에만 명시적인 overwrite 옵션을 사용한다.

업로드 스크립트가 실패한 캐릭터가 있으면 `generate-character-profile-seed.ts`를 실행하지 않는다.

## CSV 생성 규칙

### characters_seed.csv

`characters_seed.csv`는 모든 영화별 `characters.config.ts`와 캐릭터별 persona prompt Markdown을 읽어 생성한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | `movie_id + character_slug` 기반 deterministic UUID v5 |
| `movie_id` | `characters.config.ts`의 `movieId` |
| `actor_person_id` | `characters.config.ts`의 `actorPersonId`, 없으면 null |
| `name` | `characters.config.ts`의 `name` |
| `description` | `characters.config.ts`의 `description` |
| `greeting` | `characters.config.ts`의 `greeting` |
| `persona_prompt` | `{movie_id}/prompts/characters/{character_slug}.md` 본문 |
| `avatar_storage_path` | `characters.config.ts`의 `avatarStoragePath` |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

`id`는 같은 `movie_id`와 `character_slug`에 대해 항상 같은 값이 생성되어야 한다. UUID namespace는 생성 스크립트에 상수로 고정한다.

## 적재 순서

이 문서의 CSV는 캐릭터 이미지 업로드가 끝난 뒤 생성하고, 캐릭터 채팅 seed 중 가장 먼저 적재한다.

```text
1. upload-character-images.ts
2. generate-character-profile-seed.ts
3. characters_seed.csv
```

`character_chat_event_participants`와 `character_chat_default_questions`는 `characters.id`를 참조하므로 `characters_seed.csv` 적재 이후 실행한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| 초기 영화 수 | 채팅 지원 영화 2개 |
| 영화별 캐릭터 수 | 대표 캐릭터 2~4명 |
| `characters.movie_id` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `characters.actor_person_id` FK | 값이 있으면 모든 `actor_person_id`가 `people.id`에 존재 |
| 배우 누락 경고 | 명확한 배우가 있지만 `actorPersonId = null`인 경우 경고 출력 |
| 캐릭터 이름 중복 | 같은 영화 안에서 `name` 중복 0건 |
| `character_slug` 중복 | 같은 영화 안에서 생성된 slug 중복 0건 |
| `characters.id` | UUID v5 재생성 결과와 CSV의 `id`가 일치 |
| `description` | 비어 있지 않고 스포일러를 포함하지 않음 |
| `greeting` | 비어 있지 않음 |
| `persona_prompt` | 비어 있지 않음 |
| persona 섹션 | 필수 섹션 9개를 모두 포함 |
| `avatar_storage_path` | 모든 row에 존재 |
| 이미지 path 형식 | `movies/{movie_id}/characters/{character_slug}.webp` 형식 |
| 이미지 object | Supabase `character-images` bucket에 해당 object 존재 |
| timestamp 컬럼 | `created_at`, `updated_at`은 CSV에 포함하지 않음 |

# 캐릭터 기본 질문 Seed 적재 계획

## 범위

이 문서는 캐릭터 채팅 시작 시 보여줄 캐릭터별 기본 추천 질문 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `character_chat_default_questions` | `character_chat_default_questions_seed.csv` | 캐릭터별 채팅 시작 기본 질문 |

테이블 정의는 [../../db-schema/character-chat.md](../../db-schema/character-chat.md)를 기준으로 한다. 전체 캐릭터 채팅 seed 문서 구성과 전체 CSV 적재 순서는 [2.character-chat-seed-plan.md](./2.character-chat-seed-plan.md)를 따른다.

## 초기 Seed 대상

초기 기본 질문 seed는 2-1에서 선정한 채팅 지원 영화 2개와 대표 캐릭터를 대상으로 작성한다.

기본 질문은 캐릭터당 정확히 4개 작성한다. 질문은 채팅 시작 전에 사용자에게 보여주는 고정 추천 질문이며, 대화 중 다음 질문 후보는 런타임 API 응답에서 동적으로 생성한다.

`character_chat_default_questions` 스키마는 변경하지 않는다. 기본 질문은 특정 사건과 DB에서 직접 연결하지 않는다. 사용자가 기본 질문을 클릭하면 해당 질문이 첫 사용자 메시지로 전송되고, 응답 생성 로직이 캐릭터 persona와 사건/관점 데이터를 바탕으로 context를 구성한다.

## 선행 데이터

이 문서는 다음 데이터가 확정된 뒤 진행한다.

| 선행 데이터 | 이유 |
|---|---|
| `characters_seed.csv` | `character_chat_default_questions.character_id`가 `characters.id`를 참조한다. |
| 영화별 `characters.config.ts` | 질문의 `characterName`을 2-1 캐릭터 목록과 대조한다. |

기본 질문은 2-1에서 생성한 캐릭터 ID 규칙에 의존한다. 따라서 실제 검증과 적재는 `characters_seed.csv` 생성 이후 수행한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| 영화별 `default-questions.config.ts` | 캐릭터별 질문, 표시 순서, 검수용 category |
| 영화별 `characters.config.ts` | `characterName` 검증과 `character_id` 계산 |
| `characters_seed.csv` | `character_id` FK 검증 |

원천 파일 구조는 다음을 따른다.

```text
data/seeds/character-chat/movies/{movie_id}/characters.config.ts
data/seeds/character-chat/movies/{movie_id}/default-questions.config.ts
data/seeds/character-chat/generated/character_chat_default_questions_seed.csv
scripts/seeds/generate-character-default-question-seed.ts
```

사람이 관리하는 원천 데이터는 영화별 `movies/{movie_id}` 하위에 둔다. `generated/*.csv`는 스크립트 산출물이므로 직접 편집하지 않는다.

## 영화별 Default Questions Config 규칙

`default-questions.config.ts`는 해당 영화의 캐릭터별 기본 질문을 정의한다.

기본 형태:

```ts
export const characterChatDefaultQuestions = {
  movieId: 603,
  questions: [
    {
      characterName: "Neo",
      order: 1,
      category: "event",
      question: "처음 현실이 이상하다고 느낀 순간은 언제였어?",
    },
  ],
} as const
```

필수 필드는 다음과 같다.

| 필드 | 설명 |
|---|---|
| `movieId` | TMDB movie id. 2-1의 영화별 `characters.config.ts`와 일치해야 한다. |
| `questions[].characterName` | 2-1 `characters.config.ts`의 캐릭터 이름과 일치해야 한다. |
| `questions[].order` | 해당 캐릭터 안에서 표시 순서. 1부터 4까지 직접 작성한다. |
| `questions[].category` | 검수용 질문 유형. CSV와 DB에는 저장하지 않는다. |
| `questions[].question` | 사용자에게 표시할 기본 질문 문장. |

`category`는 seed 품질 검수용 필드다. `character_chat_default_questions` 테이블이나 CSV에는 포함하지 않는다.

허용 category:

| category | 캐릭터당 개수 | 설명 |
|---|---:|---|
| `event` | 2개 | 영화 속 사건, 선택, 갈등, 전환점을 묻는 질문 |
| `relationship` | 1개 | 다른 캐릭터와의 관계, 감정, 신뢰, 갈등을 묻는 질문 |
| `identity` | 1개 | 캐릭터의 성격, 가치관, 세계관, 자기 인식을 묻는 질문 |

## 질문 작성 규칙

질문은 사용자가 캐릭터에게 직접 묻는 말 형태로 작성한다.

예시:

```text
그 선택을 하기 직전에 가장 두려웠던 건 뭐였어?
그 사람을 아직도 믿고 있어?
너에게 진짜 현실이라는 건 어떤 의미야?
```

모든 질문은 한국어로 작성한다. 캐릭터 원작의 분위기를 고려하되, 영화 원문 대사나 원문 대사의 번역을 저장하지 않는다.

사용자는 영화를 이미 본 사람으로 가정하므로 질문에 스포일러성 사건이나 결말 이후 감정을 포함할 수 있다. 다만 질문은 캐릭터가 답변할 수 있는 형태여야 하며, 영화 안에서 끝까지 알 수 없는 사실을 단정적으로 요구하지 않는다.

기본 질문은 특정 사건을 떠올릴 수 있어도 DB에서 사건과 직접 연결하지 않는다. 따라서 `related_event_id`, `relatedEventOrder`는 사용하지 않는다.

## Character Slug 참조 규칙

`default-questions.config.ts`에는 `characterSlug`나 `character_id`를 직접 쓰지 않는다. 사람이 읽기 쉬운 `characterName`만 작성한다.

생성 스크립트는 2-1과 같은 slug 규칙으로 `characterName`을 정규화하고, 해당 영화의 `characters.config.ts`에 같은 이름의 캐릭터가 있는지 검증한다.

정규화된 `character_slug`는 다음 용도로만 사용한다.

```text
1. 2-1과 같은 deterministic character UUID v5 계산
2. default question UUID v5 생성 입력값
3. characterName 오타와 중복 검증
```

2-1 캐릭터 목록에 없는 `characterName`이 질문에 있으면 seed 생성은 실패한다.

## Question Slug 규칙

`question_slug`는 DB 컬럼이나 CSV 컬럼으로 저장하지 않는다. 생성 스크립트가 `question`을 정규화해 내부적으로 사용한다.

정규화 규칙은 2-1의 `character_slug`와 동일하게 적용한다.

```text
소문자로 변환한다.
공백과 특수문자는 하이픈으로 변환한다.
연속 하이픈은 하나로 줄인다.
앞뒤 하이픈은 제거한다.
같은 캐릭터 안에서 유일해야 한다.
```

같은 캐릭터 안에서 생성된 `question_slug`가 중복되면 seed 생성은 실패한다.

## CSV 생성 규칙

`character_chat_default_questions_seed.csv`는 모든 영화별 `default-questions.config.ts`를 읽어 생성한다.

### character_chat_default_questions_seed.csv

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | `movie_id + character_slug + question_slug` 기반 deterministic UUID v5 |
| `character_id` | 2-1과 같은 `movie_id + character_slug` 기반 deterministic UUID v5 |
| `question` | `questions[].question` |
| `display_order` | `questions[].order` |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

`category`는 config 검수용 필드이므로 CSV에 포함하지 않는다.

`id`는 같은 `movie_id`, `character_slug`, `question_slug`에 대해 항상 같은 값이 생성되어야 한다. UUID namespace는 생성 스크립트에 상수로 고정한다.

UUID v5 입력 문자열은 다음 형식을 따른다.

```text
character-chat-default-question:{movie_id}:{character_slug}:{question_slug}
```

## 생성 스크립트 규칙

생성 스크립트는 다음 경로에 둔다.

```text
scripts/seeds/generate-character-default-question-seed.ts
```

스크립트는 다음 작업을 수행한다.

| 단계 | 설명 |
|---|---|
| config 로드 | 모든 영화별 `default-questions.config.ts`와 `characters.config.ts`를 읽는다. |
| 캐릭터 검증 | 질문의 `characterName`이 같은 영화의 2-1 캐릭터 목록에 존재하는지 확인한다. |
| character slug 생성 | 2-1과 같은 규칙으로 `character_slug`를 계산한다. |
| question slug 생성 | `question`에서 `question_slug`를 계산한다. |
| question UUID 생성 | `character-chat-default-question:{movie_id}:{character_slug}:{question_slug}`로 UUID v5를 생성한다. |
| character UUID 생성 | 2-1과 같은 규칙으로 `character_id`를 생성한다. |
| category 검증 | 캐릭터당 `event` 2개, `relationship` 1개, `identity` 1개인지 확인한다. |
| CSV 출력 | 기본 질문 CSV를 `generated/` 아래에 생성한다. |

스크립트는 검증 실패가 있으면 CSV를 생성하지 않는다. 이미 생성된 CSV가 있더라도 실패한 입력으로 덮어쓰지 않는다.

## 적재 순서

이 문서의 CSV는 `characters_seed.csv` 생성 이후 만들고 적재한다.

```text
1. characters_seed.csv
2. generate-character-default-question-seed.ts
3. character_chat_default_questions_seed.csv
```

`character_chat_default_questions_seed.csv`는 `characters_seed.csv`를 참조하므로 `characters_seed.csv`가 먼저 준비되어야 한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| 초기 영화 수 | 2-1과 같은 채팅 지원 영화 2개 |
| 캐릭터 범위 | 2-1에서 seed한 모든 대표 캐릭터 |
| 캐릭터별 질문 수 | 정확히 4개 |
| `characterName` | 같은 영화의 `characters.config.ts`에 존재 |
| `display_order` | 캐릭터별로 1, 2, 3, 4가 중복 없이 존재 |
| `category` | `event`, `relationship`, `identity` 중 하나 |
| category 구성 | 캐릭터별 `event` 2개, `relationship` 1개, `identity` 1개 |
| 질문 형태 | 사용자가 캐릭터에게 직접 묻는 한국어 문장 |
| 질문 중복 | 같은 캐릭터 안에서 동일 질문 0건 |
| `question_slug` 중복 | 같은 캐릭터 안에서 생성된 slug 중복 0건 |
| `id` | UUID v5 재생성 결과와 CSV의 `id`가 일치 |
| `character_id` FK | 모든 `character_id`가 `characters.id`에 존재 |
| 원문 대사 | 영화 원문 대사 또는 원문 번역 저장 금지 |
| 사건 연결 컬럼 | `related_event_id`, `relatedEventOrder`는 사용하지 않음 |
| timestamp 컬럼 | `created_at`, `updated_at`은 CSV에 포함하지 않음 |

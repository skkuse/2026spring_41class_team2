# MovieLens 추천/통계 Seed 적재 계획

## 범위

이 문서는 MovieLens subset으로 확정된 영화 3,000개를 기준으로 영화 통계와 Item CF 추천 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `movie_stats` | `movie_stats_seed.csv` | 영화별 MovieLens 집계와 Cinemate 리뷰 집계 초기값 |
| `movie_similarities` | `movie_similarities_seed.csv` | 기준 영화별 Item CF 유사 영화 결과 |
| `movie_tags` | `movie_tags_seed.csv` | MovieLens Tag Genome tag master |
| `movie_tag_relevances` | `movie_tag_relevances_seed.csv` | 영화별 Tag Genome relevance 점수 |

`movie_stats`, `movie_similarities` 테이블 정의는 [../../db-schema/item-cf-recommendations.md](../../db-schema/item-cf-recommendations.md)를 기준으로 한다. `movie_tags`, `movie_tag_relevances` 테이블 정의는 [../../db-schema/recommendation-chat.md](../../db-schema/recommendation-chat.md)를 기준으로 한다. 전체 seed 문서 구성과 전체 CSV 적재 순서는 [1.movielens-subset-seed-plan.md](./1.movielens-subset-seed-plan.md)를 따른다.

## 선행 데이터

이 문서는 [1-1.movie-catalog-seed-plan.md](./1-1.movie-catalog-seed-plan.md)의 산출물인 `movies_seed.csv`가 확정된 뒤 진행한다.

`movie_stats.tmdb_id`는 `movies.id`를 참조한다. `movie_similarities.source_tmdb_id`, `target_tmdb_id`는 모두 `movie_stats.tmdb_id`를 참조하므로 `movie_stats_seed.csv`를 먼저 적재해야 한다. `movie_tag_relevances.tmdb_id`는 `movies.id`를, `tag_id`는 `movie_tags.tag_id`를 참조한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| `movies_seed.csv` | 최종 3,000개 영화의 `tmdb_id`와 `movielens_id` 매핑 |
| MovieLens `ratings.csv` | 평균 평점, 평점 수, Item CF 유사도 계산 |
| MovieLens `tags.csv` | 사용자 태그 수 계산 |
| MovieLens `genome-tags.csv` | Tag Genome tag master 생성 |
| MovieLens `genome-scores.csv` | 영화별 tag relevance 생성 |

`movie_similarities`는 온보딩 기반 Item CF 추천에 사용하고, Tag Genome seed는 AI 추천 채팅의 자연어 조건 기반 후보 조회에 사용한다.

## CSV 생성 규칙

### movie_stats_seed.csv

서비스에 노출되는 모든 `movies` row는 대응하는 `movie_stats` row를 가진다.

| 컬럼 | 생성 규칙 |
|---|---|
| `tmdb_id` | `movies.id` |
| `movielens_avg_rating` | MovieLens `ratings.csv` 기준 평균 평점 |
| `movielens_rating_count` | MovieLens `ratings.csv` 기준 평점 수 |
| `cinemate_rating_sum` | 초기값 0 |
| `cinemate_review_count` | 초기값 0 |
| `user_tag_count` | MovieLens `tags.csv` 기준 사용자 태그 수 |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

### Item CF 생성 파라미터

초기 Item CF seed는 다음 파라미터를 기준으로 생성한다.

| 파라미터 | 값 | 설명 |
|---|---|---|
| `similarityMethod` | `adjusted_cosine` | 사용자별 평점 성향을 보정한 뒤 영화 간 유사도를 계산 |
| `minCoRatingCount` | 30 | 두 영화를 모두 평가한 사용자가 30명 미만이면 제외 |
| `minSimilarityScore` | 0 | 같이 좋아하는 경향이 있는 양수 유사도만 저장 |
| `maxSimilarMoviesPerSource` | 50 | 기준 영화별 최대 50개 유사 영화 저장 |
| `storeBidirectionalCandidates` | true | 유사도 후보를 `(A, B)`와 `(B, A)` 양방향 row로 만든 뒤 기준 영화별 상위 50개를 저장 |

이 값들은 추천 품질 확인 후 조정할 수 있는 seed 생성 파라미터다. 테이블 row마다 다른 값이 아니므로 DB 컬럼으로 저장하지 않는다.

### movie_similarities_seed.csv

최종 3,000개 영화 내부에서 MovieLens rating matrix를 사용해 Item CF 유사도를 계산한다.

유사도는 단순 평균 평점이 아니라, 같은 사용자가 두 영화를 평소 성향 대비 비슷하게 선호했는지를 기준으로 계산한다. 평점을 후하게 주는 사용자와 짜게 주는 사용자의 차이를 줄이기 위해 사용자별 평균 평점을 기준으로 보정한 adjusted cosine similarity를 사용한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `source_tmdb_id` | 기준 영화의 TMDB id |
| `target_tmdb_id` | 추천 대상 영화의 TMDB id |
| `score` | Item CF 유사도 점수 |
| `co_rating_count` | 두 영화를 함께 평가한 사용자 수 |

생성 규칙:

1. 최종 3,000개 영화에 속한 MovieLens 평점만 사용한다.
2. 사용자별 평균 평점을 계산한다.
3. 각 평점을 `사용자 영화 평점 - 사용자 평균 평점`으로 보정한다.
4. 영화 쌍마다 adjusted cosine similarity를 계산한다.
5. `co_rating_count < 30`인 영화 쌍은 제외한다.
6. `score <= 0`인 영화 쌍은 제외한다.
7. 기준 영화별로 `score DESC`, `co_rating_count DESC`, `target_tmdb_id ASC` 순으로 정렬한다.
8. 유사도는 대칭이므로 후보 단계에서 `(A, B)`와 `(B, A)` 양방향 row를 만든다.
9. 기준 영화별로 상위 50개 유사 영화만 최종 저장한다.

`source_tmdb_id`와 `target_tmdb_id`는 모두 최종 3,000개 영화 안에 있어야 한다.

자기 자신을 추천 대상으로 저장하지 않는다.

### movie_tags_seed.csv

MovieLens `genome-tags.csv`의 tag 목록을 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `tag_id` | MovieLens `tagId` |
| `tag` | MovieLens `tag` |

### movie_tag_relevances_seed.csv

최종 3,000개 영화에 해당하는 MovieLens `genome-scores.csv` row만 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `tmdb_id` | `movies.id` |
| `tag_id` | MovieLens `tagId` |
| `relevance` | MovieLens `relevance` |

`tmdb_id`는 최종 3,000개 영화 안에 있어야 한다. `tag_id`는 `movie_tags_seed.csv`에 존재해야 한다.

## 적재 순서

이 문서의 CSV는 다음 순서로 적재한다.

```text
1. movie_stats_seed.csv
2. movie_similarities_seed.csv
3. movie_tags_seed.csv
4. movie_tag_relevances_seed.csv
```

`movie_similarities`는 `movie_stats(tmdb_id)`를 참조하므로 `movie_stats` 적재 이후 실행한다. `movie_tag_relevances`는 `movie_tags`를 참조하므로 tag master 적재 이후 실행한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| `movie_stats` row 수 | 3,000개 |
| `movie_stats.tmdb_id` FK | 모든 `tmdb_id`가 `movies.id`에 존재 |
| `movie_stats.tmdb_id` 중복 | 0건 |
| Cinemate 리뷰 집계 초기값 | `cinemate_rating_sum = 0`, `cinemate_review_count = 0` |
| `movie_similarities` row 수 | 최대 150,000개 |
| 기준 영화 FK | 모든 `source_tmdb_id`가 `movie_stats.tmdb_id`에 존재 |
| 대상 영화 FK | 모든 `target_tmdb_id`가 `movie_stats.tmdb_id`에 존재 |
| 자기 자신 추천 | `source_tmdb_id = target_tmdb_id` 0건 |
| 최소 공평가 수 | 모든 row의 `co_rating_count >= 30` |
| 유사도 점수 | 모든 row의 `score > 0` |
| 기준 영화별 추천 수 | 최대 50개 |
| 정렬 순서 | 같은 `source_tmdb_id` 안에서 `score DESC`, `co_rating_count DESC`, `target_tmdb_id ASC` 순서 |
| 양방향 후보 생성 | 유사도 계산 결과는 각 영화를 기준 영화로 조회할 수 있도록 양방향 후보 row를 만든 뒤 source별 상위 50개로 제한 |
| `movie_tags` 중복 | `tag_id`, `tag` 중복 0건 |
| `movie_tag_relevances` FK | 모든 `tmdb_id`가 `movies.id`에 존재 |
| `movie_tag_relevances` FK | 모든 `tag_id`가 `movie_tags.tag_id`에 존재 |
| `movie_tag_relevances` 중복 | `(tmdb_id, tag_id)` 중복 0건 |
| relevance 범위 | 모든 row의 `relevance`가 0 이상 1 이하 |

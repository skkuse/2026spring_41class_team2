# 영화 출연진/제작진 Seed 적재 계획

## 범위

이 문서는 MovieLens subset으로 확정된 영화 3,000개를 기준으로 TMDB credits 데이터를 수집해 인물, 출연진, 제작진 seed CSV를 생성하고 적재하는 계획을 다룬다.

대상 테이블:

| 테이블 | seed 파일 | 설명 |
|---|---|---|
| `people` | `people_seed.csv` | TMDB 인물 정보 |
| `movie_casts` | `movie_casts_seed.csv` | 영화별 출연진 |
| `movie_crew` | `movie_crew_seed.csv` | 영화별 제작진 |

테이블 정의는 [../../db-schema/people.md](../../db-schema/people.md)를 기준으로 한다. 전체 seed 문서 구성과 전체 CSV 적재 순서는 [1.movielens-subset-seed-plan.md](./1.movielens-subset-seed-plan.md)를 따른다.

## 선행 데이터

이 문서는 [1-1.movie-catalog-seed-plan.md](./1-1.movie-catalog-seed-plan.md)의 산출물인 `movies_seed.csv`가 확정된 뒤 진행한다.

`movie_casts.movie_id`와 `movie_crew.movie_id`는 `movies.id`를 참조하므로, credits seed는 최종 3,000개 영화에 대해서만 생성한다.

## 입력 원천

| 원천 | 사용 내용 |
|---|---|
| `movies_seed.csv` | TMDB movie id 기준 최종 영화 목록 |
| TMDB movie credits | cast, crew 목록 |
| TMDB person summary | `people` 보강 필드 |

TMDB credits 응답만으로 `people`에 필요한 필드를 채울 수 있으면 별도 person detail 조회는 하지 않는다. credits 응답에 없는 필드가 필요하면 TMDB person summary를 추가 조회한다.

## CSV 생성 규칙

### people_seed.csv

`people.id`는 TMDB person id를 그대로 사용한다. 최종 3,000개 영화의 cast와 crew에 등장한 person을 중복 제거해 저장한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `id` | TMDB person id |
| `name` | TMDB person name |
| `profile_path` | TMDB profile path, 없으면 null |
| `known_for_department` | TMDB known_for_department, 없으면 null |
| `popularity` | TMDB popularity, 없으면 null |

`created_at`, `updated_at`은 DB default를 사용하므로 CSV에 포함하지 않는다.

같은 person id가 여러 영화 또는 cast/crew 양쪽에 등장하면 `people_seed.csv`에는 한 번만 저장한다.

### movie_casts_seed.csv

TMDB credits 응답의 `cast` 배열을 기준으로 생성한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `movie_id` | `movies.id` |
| `person_id` | TMDB cast person id |
| `character_name` | TMDB cast character |
| `cast_order` | TMDB cast order, 없으면 null |

`character_name`이 비어 있는 cast row는 화면 표시 가치와 PK 안정성이 낮으므로 저장하지 않는다.

### movie_crew_seed.csv

TMDB credits 응답의 `crew` 배열을 기준으로 생성한다.

| 컬럼 | 생성 규칙 |
|---|---|
| `movie_id` | `movies.id` |
| `person_id` | TMDB crew person id |
| `department` | TMDB crew department |
| `job` | TMDB crew job |

`department` 또는 `job`이 비어 있는 crew row는 저장하지 않는다. `(movie_id, person_id, department, job)` 조합을 기준으로 중복을 제거한다.

감독 표시는 `job = 'Director'`를 기준으로 조회하므로 Director crew row는 누락되지 않아야 한다.

## 적재 순서

이 문서의 CSV는 다음 순서로 적재한다.

```text
1. people_seed.csv
2. movie_casts_seed.csv
3. movie_crew_seed.csv
```

`movie_casts`와 `movie_crew`는 `movies(id)`와 `people(id)`를 모두 참조한다. `movies`는 1-1에서 먼저 적재되어 있어야 하고, `people`은 이 문서에서 cast/crew보다 먼저 적재한다.

## 검증 기준

| 항목 | 기준 |
|---|---|
| `people` 중복 | `id` 중복 0건 |
| `people.name` 누락 | 0건 |
| `movie_casts` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `movie_casts` FK | 모든 `person_id`가 `people.id`에 존재 |
| `movie_casts` 중복 | `(movie_id, person_id, character_name)` 중복 0건 |
| `movie_crew` FK | 모든 `movie_id`가 `movies.id`에 존재 |
| `movie_crew` FK | 모든 `person_id`가 `people.id`에 존재 |
| `movie_crew` 중복 | `(movie_id, person_id, department, job)` 중복 0건 |
| 감독 row | 감독이 있는 영화는 `job = 'Director'` row 존재 |
